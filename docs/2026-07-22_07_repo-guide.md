## Prerequisites

- **Python 3.12** (from `pyproject.toml`'s `requires-python = ">=3.12"`)
- **uv** (package manager): `curl -LsSf https://astral.sh/uv/install.sh | sh`
- **CUDA 12.6** compatible GPU + drivers
- **~30 GB** disk for a full rollout setup (CUDA + Isaac Sim + deps)

## Quick Start (Main Venv — Training/Inference)

This is all you need for training and policy inference (no Isaac Sim).

```bash
# Clone with submodules
git clone --recurse-submodules https://github.com/your-org/probenet
cd probenet

# Create main venv with all deps (openpi, torch 2.7.1, jax, flax, ...)
uv sync --frozen --no-dev

# Verify
uv run python -c "import openpi; import probenet"
```

Key deps (`pyproject.toml`):
- `torch==2.7.1` — pinned by both probenet and openpi
- `jax[cuda12]==0.5.3`
- `openpi` — local editable install from `backends/openpi/`
- `flax==0.10.2`, `orbax-checkpoint==0.11.13`, `transformers==4.53.2`

## Isaac Sim Setup (Rollout — Simulation)

Rollout needs Isaac Sim, which requires a separate virtual environment due to
conflicting torch versions (openpi needs torch 2.7.1, Isaac Sim 6.0.1 needs
torch 2.11.0). They never share a Python process — the policy runs as a
WebSocket server, the Isaac Sim worker connects to it.

```bash
# Create isolation venv for Isaac Sim
uv venv .venv-isaac

# Install Isaac Sim with its required torch
uv pip install --python .venv-isaac --upgrade torch==2.11.0 \
    --index-url https://download.pytorch.org/whl/cu128
uv pip install --python .venv-isaac \
    "isaacsim[all,extscache]==6.0.1.0" \
    --extra-index-url https://pypi.nvidia.com \
    --index-strategy unsafe-best-match --prerelease=allow

# Install probenet code (no deps — isaacsim pins its own)
uv pip install --python .venv-isaac --no-deps -e .

# Accept EULA and verify
export OMNI_KIT_ACCEPT_EULA=YES
.venv-isaac/bin/python -c "import isaacsim; import probenet"
```

### Architecture — Two Processes

```
┌─────────────────────┐     WebSocket      ┌──────────────────────────┐
│  .venv (torch 2.7.1) │ ◄──────────────► │  .venv-isaac              │
│                     │                   │  (torch 2.11.0)          │
│  openpi policy      │                   │  Isaac Sim rollout       │
│  server             │                   │  worker                  │
└─────────────────────┘                   └──────────────────────────┘
```

The policy server (`backends/openpi/src/openpi/serving/websocket_policy_server.py`)
and Isaac Sim worker are started as separate processes.

## Why Two Virtual Environments?

| Venv | Torch | Contains | Purpose |
|---|---|---|---|
| `.venv` | 2.7.1 | openpi, jax, flax, probenet | Policy inference server |
| `.venv-isaac` | 2.11.0 | isaacsim, isaaclab, probenet (no deps) | Isaac Sim simulation |

They are separate because:
1. **openpi** pins `torch==2.7.1` (its lockfile requirement)
2. **Isaac Sim 6.0.1.0** requires `torch==2.11.0`
3. They communicate over WebSocket — no Python process needs both

## Docker (Optional, Disabled in CI)

A multi-target Dockerfile is at `docker/Dockerfile` but is **disabled in CI**
(the workflow only runs on `workflow_dispatch:`) because the ~30 GB rollout
image exceeds GHA standard runner disk space.

To build locally:

```bash
docker build --target trainer -t probenet:trainer -f docker/Dockerfile .
docker build --target rollout -t probenet:rollout -f docker/Dockerfile .
```
