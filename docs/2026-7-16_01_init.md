# Project: ProbeNet
*(working title — alt: TactilePrior, VisTac-Cond)*

Vision-informed interactive perception for physics-aware manipulation.
Inspired by RehabNet's patient-parameter conditioning: instead of conditioning
a policy on patient state (ROM, spasticity, pain, fatigue) to generalize
across a spectrum of patients, ProbeNet conditions a manipulation policy on
object physical-parameter state (mass, friction, compliance) to generalize
across a spectrum of objects with similar appearance but different physics.

## Pipeline

1. **Visual Perception**
   Vision encoder observes the object → visual params (shape, size, gloss,
   coarse material class).

2. **Vision-Informed Interactive Perception (Probe Phase)**
   Visual params set the probe's initial strategy (e.g. glossy surface →
   assume lower friction prior → apply firmer initial grip / more cautious
   force ramp during probe).
   Scripted probe (squeeze + lift-hold) executes, reading servo
   current/torque → physical params (mass, compliance, true friction).
   This step can *confirm or correct* the visual prior — the interesting
   result is when it corrects it (e.g. matte-but-slippery object).

3. **Manipulation Policy**
   Conditioned on (visual params + physical params) via a small parameter
   graph / GNN, feeding a BC or ACT action head.
   Executes task (pick-and-place) with physics-aware trajectory and force.

## Baseline vs. Contribution
- **Baseline:** BC/ACT trained on demos, vision-only, no probe.
- **+Vision-informed probe conditioning (ProbeNet):** same backbone,
  added visual+physical parameter graph conditioning.

## Headline Experiment
Two visually near-identical objects (e.g. matte fragile shell vs.
weighted/slippery shell) with opposite correct grip-force/trajectory
requirements. Baseline fails asymmetrically (crushes one, drops the other).
ProbeNet succeeds on both by resolving the ambiguity through probing.

## Lineage
Architecturally inspired by RehabNet (thesis project): parameter-set
conditioning for generalization across a subject spectrum, ported from
patient-state conditioning (clinical domain) to object-physical-state
conditioning (manipulation domain).

## Hardware
- SO-101 leader-follower arm pair (teleop already working)
- Static-mounted RealSense D435if (third-person view, RGB stream;
  depth optional stretch goal)
- Servo current/torque readout used as the probe signal source
  (no added hardware — same signal for compliance check and
  lift-and-hold mass/slip check)

## Object Set
- Shell A (fragile): thin-wall PLA print, hollow, low mass
- Shell B (rigid/heavy/slippery): thick-wall or weighted-core print,
  smoother surface finish
- Same external geometry (~5-6cm sphere), painted to match, so
  visual params alone cannot fully disambiguate them —
  physical params must carry the residual signal

## 4-Week Plan

**Week 1 — Setup + Data**
- Fabricate Shell A / Shell B (print, weight/fill, paint to match)
- Calibrate probe: squeeze force thresholds (compliance signal) and
  lift-hold height/duration (mass/slip signal) via servo current logging
- Collect clean pick-and-place demos on both shells (teleop, ~15-20 each)
- Log probe readings alongside each demo run
- Stand up minimal Hugo project page (live URL early,
  iterate later)

**Week 2 — Baseline**
- Train BC/ACT on demos only (vision + proprioception → actions)
- Evaluate baseline separately on Shell A and Shell B
- Confirm asymmetric failure (crush vs. drop) — this is your control result

**Week 3 — ProbeNet Conditioning**
- Build vision encoder → visual param embedding (shape/size/gloss)
- Build parameter graph (visual nodes + probe-derived physical nodes) + GNN
- Condition manipulation policy on graph embedding, retrain
- Wire vision-informed probe strategy (gloss → initial force prior)

**Week 4 — Evaluation + Packaging**
- Run baseline vs. ProbeNet on both shells, record success/failure table
- Held-out check: does probe correct a misleading visual prior on at least
  one trial (i.e. gloss suggested one thing, probe revealed another)?
- Package repo: README (this spec), results table, rollout clips,
  citations positioning against interactive-perception literature
  (Dutta et al. visuo-tactile framework, tactile-VLM property inference,
  general interactive perception survey)
- Finalize project page with real results/videos
